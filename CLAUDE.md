# CLAUDE.md — PromptCanvas (프로젝트 룰 / 우리 반 알림장)

> 이 파일은 PromptCanvas 작업을 이어갈 때 가장 먼저 읽는 "결정 기록"입니다.
> 글로벌 룰(`~/.claude/CLAUDE.md`)을 따르고, 여기엔 이 프로젝트만의 약속을 적습니다.

## 1. 이게 뭐냐 (한 줄 정의)

**카드뉴스·썸네일·상세페이지·배너·인스타 포스트** 같은 디자인을 만들 때,
옵션만 골라서 **AI용 디자인 프롬프트**를 뽑아주는 도구. (= "디자인 프롬프트 빌더")

## 2. 누가 쓰나

- **형석님** (비개발자, AI로 빠르게 학습 중) 본인 + **가까운 사람들과 공유**.
- 로그인·회원가입 없는, 브라우저로 바로 여는 **정적 웹앱**.

## 3. 왜 만드나 (해결하는 불편)

> AI로 디자인 뽑을 때 — ① 프롬프트가 매번 막막하고 ② 반복 타이핑이 귀찮고
> ③ 원하는 느낌이 안 나온다. → **더 빠르게, 더 좋은 결과를 뽑고 싶다.**

핵심 통찰: **AI 디자인 품질의 90%는 "프롬프트가 얼마나 좋으냐"에서 갈린다.**
그래서 이 앱의 진짜 무기는 "좋은 프롬프트를 자동으로 만들어주는 것".

## 4. 확정된 방향 — 하이브리드 (2026-06-27 결정)

```
[웹앱] 옵션 고르기 → 똑똑한 프롬프트 자동 생성 → 복사          ← 여기까지 비용 0
   ↓ (붙여넣기)
[Claude] 프롬프트 받아 실제 디자인(HTML/CSS) 생성
   ↓ (자동 변환)
[Adobe Express / Canva] 진짜 편집 가능한 결과물 → 직접 수정    ← 수정도 비용 0
```

**왜 이 방향인가:**
- "실제 결과물까지" + "공유" + "비용·서버 0"을 동시에 만족.
- 서버·API키·로그인 없이, Claude 환경에 붙은 Adobe/Canva 연결을 활용.
- 검증 완료: 카드뉴스 데모 1개를 실제 Express 문서로 변환 성공 (2026-06-27).

**안 가는 방향 (지금은):**
- Next.js + Supabase + 이미지 생성 API 같은 본격 SaaS (8주짜리, 너무 무거움).
- 웹앱 안에 이미지 생성 API 직접 붙이기 (API키·비용·서버 부담).

## 5. 비용(토큰) 원칙

| 단계 | 비용 |
|---|---|
| 웹앱에서 프롬프트 뽑기 | 🟢 0 (AI 안 씀) |
| Claude로 디자인 1개 생성 | 🟡 여기서만 사용량 듦 (디자인 1개 ≈ 대화 몇 번) |
| Express/Canva에서 수정 | 🟢 0 (직접 편집) |

- 형석님은 **정액 구독** = 토큰당 과금이 아니라 **사용량 한도** 개념.
- 절약 팁: 실험은 웹앱에서(공짜) → 확정될 때만 Claude로 뽑기 → 수정은 Express에서 직접.
- 도구 설명서(스킬) 로딩은 **세션당 1회**라 두 번째 디자인부터 가벼움.

## 6. 기술 / 작업 방식

- **단일 `index.html` 정적 웹앱.** 빌드 없음. 더블클릭으로 열림.
- 로컬 `file://` 환경이라 **fetch 금지** (CORS 차단). 데이터는 HTML/JS에 인라인.
- 외부 의존성 최소화. 폰트는 Google Fonts CDN 정도만.
- 70% 원칙: 완벽보다 빠른 실행 → 조금씩 고도화.
- 변경 후엔 브라우저로 열어 직접 확인.

```
promptcanvas/
├── index.html              ← 메인 작업본 (디자인 프롬프트 빌더)

├── 다음에-이어서.md         ← 중단 지점 메모
├── docs/                   ← 기획서(PRD), 기획 메모

│   ├── archive/            ← 이전 버전 백업
│   └── reference/          ← 참고 이미지
```
